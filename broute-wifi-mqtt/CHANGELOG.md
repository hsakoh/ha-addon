# CHANGELOG

## v0.0.5 - 2026-07-10
- entity_id が名前の音訳から自動生成され読みにくい値になっていた問題を修正（`default_entity_id` を明示指定）
- 既存のエンティティの entity_id は自動では変わりません。対処方法は [リリースノート](https://github.com/hsakoh/broute-wifi-mqtt/releases/tag/v0.0.5) を参照してください

## v0.0.4 - 2026-07-10
- MQTT へのステータス通知が毎回2回発行されていた問題を修正
- 起動時の GET プロパティ一括取得を1フレームにまとめ、起動時間を約58秒から約21秒に短縮
- インスタンスリスト通知を送らない他の ECHONET Lite ノードに対して、メーター検出で30秒待機していた問題を修正
- ログの重複・誤表記を修正（プロパティマップ読み取り、読み出し先の機器名）

## v0.0.3 - 2026-07-09
- スマートメーターのデバイス検出が完了せず無限に待機し続ける問題を修正
- インスタンスリスト通知要求(INF_REQ)を送信し、メーターの応答を確実に得るよう改善
- インスタンスリスト通知が重複して届いた際に、機器オブジェクトが二重生成される問題を修正
- 新オプション DiscoveryTimeout（デフォルト30秒）を追加。メーター検出の待機に上限を設定できます

## v0.0.2 - 2026-05-11
- プロパティ読み出し間隔を設定可能にし、ログを改善

## v0.0.1 - 2026-05-10
- Initial release
